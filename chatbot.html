Project Overview

Background: This project was a group assignment related to vibe coding and a health challenge, this being immunisation reminders. I was assigned to create a backup prototype.

Problem: Parents residing in South Africa need a reliable way to help track but also to remind them about their children's immunisation dates to manage all of their children's health, while accessing reliable information in one place.

Solution: Develop a web application to assist in immunisation journeys, with an intelligent chatbot to provide instant and reliable answers to streamline the immunisation process.

Key Features

  • Intelligent Chatbot - Powered by Gemini 2.5 Flash API for natural language understanding
  • Immunisation tracker and reminder - Reminder dates are setup automatically according to South Africa's immunisation schedule once the user selects their child's birthdate. Additionally, all vaccines including the ages children must get it is listed in a journey format, indicating what is completed, due next and completed
  • Management - Allows users to manage multiple children profiles
  • Multilingual - Provides accessibility to multiple citizens. Current languages include English, Afrikaans, isiXhosa and isiZulu
  • Text-to-Speech (TTS) - Enables users to listen to chatbot responses, improving accessibility for users who prefer audio or have reading difficulties
  • Clinic Locator - Google Maps embedded into the site leading users to another window (Google Maps) displaying nearby health facilities only
  • Community and Education Dashboard - To provide a safe space for users and to educate them. Currently this section is not fully functional and reliable for information

System Setup

System Architecture

Business Layer

  • Core Goal: To enable caregivers to manage their family immunisation journeys
  • Health Tracking: Track schedules, progress, and upcoming appointments
  • AI Assistance:: Offer an AI-powered chatbot for general health questions
  • Community & Resources: Connect parents to local clinic resources and provide articles for education

    • Application Layer

  • Next.js Frontend: Web application that users interact with. It includes all UI components, pages, and client-side logic
    • UI Components: Reusable elements like the dashboard, vaccine lists, etc
    • Page Routing: Handles navigation between different sections and pages
  • Genkit AI Flows: Set of server-side functions that orchestrate AI model interactions
    • Chatbot Flow: Manages the logic for receiving a user's question, prompting the AI model, and structuring the response
    • Summariser Flow: Handles the logic for summarising health articles
    • TTS Flow: Converts text responses from the chatbot into audible speech
  • Firebase Services: Backend services that support the application
    • Authentication Service: Manages user identity, sign-up, and login
    • Firestore Service: Handles all database operations (reading and writing user/child data)

    Tech Layer

  • Core Framework: Next.js / React (with TypeScript)
  • Frontend UI & Styling:
    • React: For building the UI
    • Tailwind CSS: For all styling
    • ShadCN UI: For the core component library (buttons, cards, dialogs)
    • Recharts: For the dashboard progress chart
  • AI & Machine Learning:
    • Genkit: The framework for defining and running AI flows
    • Google Gemini Models: The specific Large Language Models used for chat, summarisation, and TTS
  • Backend & Infrastructure:
    • Firebase SDK: The client-side library used in the Next.js app to communicate with Firebase services
    • Firebase App Hosting: The target platform for deploying the application

    Data Layer

  • Firestore Database:
    • children collection: Stores documents for each child, containing their name, date of birth, associated user ID, and a map of completed vaccines
    • Security Rules: Firestore Security Rules (managed by Firebase), ensuring that users can only access and modify their own data
  • Firebase Authentication:
    • User Store: Manages user accounts, including UID, email, display name, and authentication providers (email/password)
  • Application Data (Static):
    • src/lib/locales/*.json Contains all the internationalization (i18n) strings for different languages
    • src/lib/data.ts Holds static content like the educational articles and initial community posts
    • src/lib/vaccine-schedule.ts Contains the business logic and data for the entire South African immunization schedule
    • src/lib/placeholder-images.json Manages data for placeholder images used throughout the app

    Project Reflection

    Although vibe coding appears easy, it is really a challenge. For instance, prompting Firebase to generate certain images like a cartoon family illustration was a challenge because no matter what you prompt, it would not generate the image you want and will rather generate a realistic landscape image. Although amusing now, it was quite frustrating at the time.

    Despite this setback, Firebase allows users to prompt endlessly, allowing me to create most of the functions I wanted with easy integration such as the TTS, Gemini API, translation, a database, etc. Additionally Firebase integration is easy as it is part of Google, enabling users to integrate multiple functions and services. However. I encountered challenges accessing the free trial as I never received the verification code, essentially, limiting some features I wanted to add such as the Google Maps API integration, etc.

    For future enhancements, I would integrate more features such as:

  • Speech-To-Text
  • More languages
  • Integrate with government health systems for improved efficiency
  • Direct contact details for medical professionals
  • Verified educational health articles from trusted sources
    • Despite the navigation challenges and working primarily with generated code, I plan to deepen my understanding of the system architecture, connections, and underlying code.

    But to conclude, Firebase provides functional solutions and easy integration, though the aesthetic results don't quite match platforms like Lovable (I also created a prototype on Lovable which is not fully functional, but it can be viewed here: View Prototype) So although AI is flawed and has its limitations, it still assists us in providing a foundation that can be further built upon.
