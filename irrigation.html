Project Overview

Key Requirements: To develop an irrigation system using low-cost and low-energy IoT technologies and software development practices to create tangible improvements in greenhouse and crop development operations at ARC. The system should automate irrigation processes based on real-time sensor data and track water usage to optimise resource consumption. Furthermore, the system should include scheduling irrigation cycles, monitoring water flow rates, etc. The system must integrate with the data management and soil moisture team

Solution: Create an irrigation system that can be controlled by a mobile device to automate soil irrigation based off soil moisture, to schedule irrigation sessions and to manually activate and deactivate the irrigation. Additionally, the solution will provide analytical data to provide insights for the farm managers

Additional: This was a group assignment where all members worked both on documentation and hardware setup. My primary role was creating the frontend with another member finalising it

**Note: All information provided is based off the work I did and key information

Key features

  • Control Modes:
    • Automation - Pump automatically switches on and off based on soil moisture thresholds set (user can set thresholds in settings)
    • Scheduling - Users can create, edit and delete schedules for irrigation cycles
    • Manual - Users can manually turn on and off the pump when needed
  • System Status:
    • Pump status - Indicates if pump is on or off. Additionally if pump is on, the System Status card turns green, and when off, the card is grey
    • Countdown- A countdown runs when the user manually turns on the pump
    • Control Mode Indicator - Indicates which control mode switched the pump on
  • Key Metrics - Displays soil moisture levels, flow rate, total volume and cycle usage
  • History Page - Displays key information of the irrigation cycle that took place such as the actual duration vs its set duration and its reason, including the volume
  • Analytics Page - Displays charts that can be filtered to different time ranges so the user can use the data for insights and decision-making. Example, comparing specific months across the years, the user can identify seasonal trends, where viewing data for today can assist the user in detecting irregular patterns quickly, e.g., if the flow rate changed drastically, the user can detect that it might be faulty within that day
  • Settings Page - Gives the user the option to have control over the app, like changing the language or metric system, enabling and disabling notifications and alerts or adjusting moisture thresholds. Additionally, there is a user guide to further assist the user

    Project Reflections

    The main challenge we faced as a group was limited communication with the data management team, who weren't consistently on campus. Since our system was highly dependent on their API integration, this created coordination difficulties. However, we didn't let this stop us and continued working where we could. For future enhancements I would love to incorporate:

  • various languages to support the diverse country
  • integrate an ultrasonic sensor into the irrigation system to measure the tank level which will be displayed in the application. This will alert the farm managers about the current tank levels
  • integrate notifications to display within the users phone system through Firebase Cloud Messaging or OneSignal. This will enable users to receive notifications like pump status and duration on their devices without having to access the app.
  • integrate the weather into the app through an API. This can assist in scheduling based on weather conditions and recommendations that the app will make
  • add a manual reset for total volume
