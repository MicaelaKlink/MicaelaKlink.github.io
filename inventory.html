Project Overview

Problem: TechInnovators Inc., founded in 2010, is an engineering firm that designs and develops Internet of Things (IoT) devices and equipment. Due to the rapid growth in the company, TechInnovators Inc. encountered obstacles such as difficulties in managing the inventory, including its movements. Due to these challenges, TechInnovators Inc. requires a system that will allow for admin to manage inventory in one central place.

Key Requirements: The system that must be developed in Oracle APEX should allow for the admin to track, update, view and assign inventory. Additionally, the admin should be able to analyze and gather trends and insights about the inventory, such as how long the equipment is in use and what equipment is used the most to ensure and assist admin in better tracking of the movement of inventory and improve decision making in regards to obtaining equipment from suppliers, etc. Additionally, employees require a system that will allow them to view their historical usage and to check in their equipment via QR code scanning. The development for the employees system must be developed using Python.

Solution: Develop a system on Oracle APEX for admin to track, assign and update inventory. Additionally, develop another system using Python for employees to view inventory history and for returning inventory by scanning a QR code.

Key Features

Admin

  • Status - A quick display for admin to view current pending vs approved requests
  • Manage Inventory - Admin can add new inventory, update existing inventory and assign inventory
  • Track Inventory - Charts, reports and cards for admin to track movement of inventory
  • Manage Employees - Admin can add new employees and view existing employees inventory history
  • Analytics - Charts and Reports displayed to assist in planning and decision-making

    • Employees

  • History - Employees can view their inventory history
  • Return Inventory - Employees must return items via QR code scanning
  • Request Inventory - Employees can request inventory that admin must approve

    • Additional

  • Input - User's can type their inputs instead of scrolling through heaps of items

    • System Setup

    Database Schema (ERD)

    History:

  • BOOKING_ID: Uniquely identifies the assigned item.
  • ITEM_ID: A foreign key that assists admin in viewing which item was borrowed
  • EMPLOYEE_ID: A foreign key that assists admin in identifying who has the equipment
  • DATE_BOOKED: The date the admin assigns the item and employee obtains it
  • DATE_RETURNED: The date the employee returns the item
  • QUANTITY_ASSIGNED: The quantity of the item the admin assigns to an employee/quantity the employee receives
  • REQUEST_ID: Foreign key that links to request table

    • Employees:

  • EMPLOYEE_ID: Uniquely identifies employee
  • NAME: The name of the employee
  • SURNAME: The surname of the employee
  • DEPARTMENT: The Department the employee belongs to

    • Inventory:

  • ITEM_ID: Uniquely identifies the item/equipment/inventory
  • ITEM_NAME: The Full name of the item/equipment/inventory
  • QUANTITY: The quantity for the item
  • LAST_UPDATED: The date the inventory was last updated

    • Requests:

  • REQUEST_ID: Uniquely identifies request
  • ITEM_ID: A foreign key that links to the inventory table
  • EMPLOYEE_ID: A foreign key that links to the employee table
  • QUANTITY_REQUIRED: The quantity of an item the employee requests
  • REQUEST_DATE: The date the employee submitted the request
  • REQUIRED_DATE: The date the employee needs the item by
  • STATUS: To display Pending and Approved

    • Technology Stack and Libraries

    Core Frameworks:

  • Python: Primary language for the development for the employees frontend
  • Oracle APEX: To develop admin frontend, database purposes and the creation of the APIs
  • Streamlit: For creating interactive web applications frontend

    • Libraries & Dependencies:

  • http.client: Handles HTTP requests.
  • dotenv: Manages and load environment variables from .env file
  • base64: For encoding credentials for Basic authentication
  • cv2 (OpenCV): For webcam access for QR code scanning
  • pyzbar: For decoding QR code
  • pandas: For data manipulation and to display tables in the Streamlit app
  • urllib.parse: For URL parsing and encoding query parameters
  • json: For handling JSON data

    • Error Handling

  • If an employee attempts to use another employee's QR code to return an item, the API validates the employee ID and sends a response indicating that the item is not assigned to them
  • If an employee attempts to return an item that was already returned, the system notifies them that the item was previously checked in. The original return date remains unchanged

    • API Documentation

    The API bridges Python and Oracle APEX so data is connected and updated on both ends for both the admin and employee. It also ensures that the database in Oracle APEX is updated with each request and response made. Furthermore, the API is exposed via Oracle REST Data Services (ORDS) and secured by Basic Authentication.

    Methods (HTTPS)

  • GET - There is one GET endpoint which enables employees to retrieve their inventory history that is stored in Oracle APEX database
  • POST - There are two POST endpoints. One enables employees to insert their BOOKING_ID and scan a QR code that is linked to their EMPLOYEE_ID. Once the user executes this task, the date and the item returned will be updated in Oracle APEX database. The other endpoint enables employees to request inventory
